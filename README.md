Think of GitHub as a combination of a **cloud storage drive** (like Google Drive) and a **time machine** specifically designed for code.
As a beginner, you’ll primarily use it for these four things:
### 1. Version Control (The "Time Machine")
When you’re coding, it’s common to break something that was working five minutes ago.
 * GitHub tracks every single change you save (called a **Commit**).
 * If you make a mistake, you can look back at your history and restore the exact version of the code from yesterday or last week.
### 2. A Digital Portfolio
Since your brother has a PhD in CS, he knows that a GitHub profile is a developer's "living resume."
 * You can host your projects in **Repositories** (folders for your code).
 * It shows potential employers—and family—that you are actively learning and writing clean, documented code.
### 3. Collaboration & Feedback
GitHub is the world's largest social network for programmers.
 * **Pull Requests:** You can ask someone (like your brother) to review your code. He can leave comments on specific lines, suggesting a better way to write a function.
 * **Issues:** You can keep a "To-Do" list for your project right next to the code, tracking bugs or features you want to add.
### 4. Free Web Hosting
As you saw in your course, **GitHub Pages** is a standout feature for beginners.
 * It takes your code and turns it into a real, live website (URL) for free.
 * This is much easier than renting a server or dealing with complex web hosting companies.
### Key Terms to Know
| Term | What it actually means |
|---|---|
| **Repository (Repo)** | Your project's main folder. |
| **Commit** | A "Save Point" in your project's history. |
| **Push** | Sending your local changes up to the GitHub cloud. |
| **Pull** | Grabbing the latest changes from the cloud to your computer. |
